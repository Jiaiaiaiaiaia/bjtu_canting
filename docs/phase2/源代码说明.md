## 2. 各文件职责

### 2.1 `app.py`
- Flask 应用工厂 `create_app()`，注册蓝图、初始化数据库、绑定模板和静态目录到 `frontend/`
- 默认监听 `0.0.0.0:5001`（macOS 5000 被 AirPlay 占用）

### 2.2 `api/routes.py`
- 提供以下接口（详见 5.x 节）：
  - `POST /api/config`（参数校验后初始化引擎）
  - `POST /api/simulation/start`（带防重启动守卫）
  - `GET  /api/simulation/step`（单步推进）
  - `GET  /api/simulation/status`
  - `POST /api/simulation/pause`
  - `POST /api/simulation/finish`（fast-forward 到结束并落库）
  - `POST /api/simulation/reset`
  - `GET  /api/statistics`
  - `GET  /api/history/configs`（历史配置列表 + 汇总）
  - `GET  /api/history?config_id=N`（指定配置的快照时间序列）
- 持久化采用 SQLite：50 条快照一刷或仿真结束时刷盘

### 2.3 `simulation/engine.py`
- `Student` / `Event` / `SimulationEngine`
- 三类事件：`arrival` / `service_end` / `eat_end`
- 关键方法：
  - `start()` — 幂等，预生成所有 `arrival` 事件
  - `step()` — 推进一个事件并更新峰值统计
  - `get_statistics()` — 计算平均等待 / 服务 / 就餐时间、座位利用率、按分钟聚合的时间序列

### 2.4 `simulation/queue_sim.py`
- `Window`：窗口排队队列、当前正在打饭的学生、累计服务数
- `pick_shortest_window(windows)` — 最短队列分配策略（含正在打饭的人）
- `sample_serve_time(avg)` — 正态分布，下限 1 秒

### 2.5 `simulation/dining_sim.py`
- `Seat`：状态、占用学生、`eat_end_time`（动态计算 remaining_time 用）
- `pick_nearest_seat(seats, window_id, window_count)` — 按窗口位置就近选择
- `sample_eat_time(avg_minutes)` — 正态分布，下限 60 秒

### 2.6 `tests/`
- 4 个测试文件 + conftest，38 条用例
- `python -m pytest tests/` 0.4 秒跑完

## 3. 运行方式

### 3.1 准备环境
```bash
cd /path/to/Canteen
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install pytest    # 仅运行单元测试时需要
```

### 3.2 启动后端服务
```bash
cd backend
python app.py
# Flask 监听 http://0.0.0.0:5001
```

浏览器访问 `http://127.0.0.1:5001/` 即可看到完整界面。

### 3.3 跑单元测试
```bash
cd backend
python -m pytest tests/ -v
# 预期：38 passed in ~0.4s
```

## 4. 数据库

首次启动时自动创建 `database/simulation.db`，含两张表：
- `simulation_config`：每次提交配置入一行
- `simulation_snapshot`：每个仿真事件入一行（`config_id` 关联）

不需要手工执行任何 SQL。如需重置，删除 db 文件即可。
