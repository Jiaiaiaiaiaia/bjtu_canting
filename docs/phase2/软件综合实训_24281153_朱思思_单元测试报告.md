## 5. 通过测试发现并修复的问题

测试驱动暴露了 5 处真实缺陷（详见沟通交流记录第 3 次会议），全部修复并补对应回归用例：

| 问题 | 触发用例 | 修复点 |
|------|----------|--------|
| `total_arrived` 在 `start()` 时被一次性预统计 | `test_total_arrived_zero_right_after_start` | 计数移到 `_handle_arrival()` |
| `start()` 可重复调用导致事件队列翻倍 | `test_idempotent_start` / `test_repeated_start_does_not_pollute` | 加 `_is_started` 守卫 |
| "结束仿真" 只是 pause，统计不完整 | `test_finish_returns_complete_stats` | 新增 `/simulation/finish` fast-forward 接口 |
| 座位 `remaining_time` 是入座时锁定的静态值 | `test_remaining_time_decreases_for_same_seat` | 改用 `eat_end_time`，按 `current_time` 动态换算 |
| 缺历史配置列表接口 | `test_history_configs_endpoint` | 新增 `GET /api/history/configs` 含汇总 |
